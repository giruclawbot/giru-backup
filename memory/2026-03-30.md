## Architecture Gotchas (important for next session)

1. **Static export + dynamic routes**: always use `useParams()` / `useSearchParams()` client-side to read IDs — never trust server `params` prop in `output: 'export'` mode
2. **Nutrition always per-100g** — normalize on write: `val_per_100g = (val / servingSize) * 100`
3. **Firestore rules deploy** — `firebase deploy --only hosting` does NOT deploy rules/indexes. Use `--only hosting,firestore`
4. **New Firestore query** with `where + orderBy` on different fields = needs composite index in `firestore.indexes.json`
5. **Firebase lazy init** — never `initializeApp` at module level, always lazy with `typeof window !== 'undefined'` guard
6. **Firestore undefined fields** — `setDoc` rejects objects with `undefined` values — strip them before writing
7. **seed scripts use CLI refresh token** — `~/.config/configstore/firebase-tools.json` → `.tokens.access_token` / `.tokens.refresh_token`
8. **pdf-parse v2 API** — `new PDFParse({ data: buffer }).getText()` not old `pdfParse(buffer)` API
9. **Promise.all vs allSettled** — use `allSettled` when combining USDA + Firestore community search so one failure doesn't hide results
